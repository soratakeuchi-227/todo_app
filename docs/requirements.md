# 要件定義書

## 1. プロジェクト概要

### 1.1 プロジェクト名
Todo App（タスク管理アプリケーション）

### 1.2 目的
日常のタスクを効率的に管理し、スケジュールを可視化することで、ユーザーの生産性向上を支援する。

### 1.3 対象ユーザー
- 個人のタスク管理を行いたいユーザー
- 日々のスケジュールを整理したいユーザー

---

## 2. 機能要件

### 2.1 ユーザー管理機能

| ID | 機能名 | 説明 |
|----|--------|------|
| U-001 | ユーザー登録 | ユーザー名、メールアドレス、パスワードで新規アカウントを作成 |
| U-002 | ログイン | ユーザー名とパスワードで認証 |
| U-003 | ログアウト | セッションを終了 |
| U-004 | ログイン状態保持 | 「ログイン状態を保持」オプションでセッションを維持 |

#### バリデーション要件
- ユーザー名: 3〜80文字、一意
- メールアドレス: 有効な形式、一意
- パスワード: 6文字以上

### 2.2 タスク管理機能

| ID | 機能名 | 説明 |
|----|--------|------|
| T-001 | タスク作成 | タイトル、説明、優先度、期限、カテゴリを設定してタスクを作成 |
| T-002 | タスク編集 | 既存タスクの内容を変更 |
| T-003 | タスク削除 | タスクを削除（サブタスクも連動削除） |
| T-004 | タスク完了切替 | タスクの完了/未完了を切り替え |
| T-005 | タスク一覧表示 | フィルタリング機能付きでタスクを一覧表示 |
| T-006 | タスク詳細表示 | タスクの詳細とサブタスクを表示 |

#### タスク属性
- タイトル: 必須、200文字以内
- 説明: 任意、テキスト形式
- 優先度: 高(1)・中(2)・低(3)の3段階
- 期限: 任意、日付形式
- カテゴリ: 任意、ユーザー定義

#### フィルタリング機能
- 全て表示
- 未完了のみ表示
- 完了済みのみ表示
- カテゴリ別表示

### 2.3 サブタスク機能

| ID | 機能名 | 説明 |
|----|--------|------|
| S-001 | サブタスク作成 | 親タスクにサブタスクを追加 |
| S-002 | サブタスク編集 | サブタスクの内容を変更 |
| S-003 | サブタスク削除 | サブタスクを削除 |
| S-004 | サブタスク完了切替 | サブタスクの完了/未完了を切り替え |
| S-005 | 進捗表示 | 親タスクにサブタスクの進捗率を表示 |

### 2.4 カテゴリ管理機能

| ID | 機能名 | 説明 |
|----|--------|------|
| C-001 | カテゴリ作成 | カテゴリ名と色を設定して作成 |
| C-002 | カテゴリ編集 | カテゴリ名・色を変更 |
| C-003 | カテゴリ削除 | カテゴリを削除（タスクのカテゴリは解除） |
| C-004 | カテゴリ一覧表示 | 登録済みカテゴリを一覧表示 |

#### カテゴリ属性
- カテゴリ名: 必須、50文字以内
- 色: HEXカラーコード（デフォルト: #6c757d）

### 2.5 スケジュール管理機能

| ID | 機能名 | 説明 |
|----|--------|------|
| SC-001 | タスクのスケジュール設定 | タスクに予定日・開始時刻・終了時刻を設定 |
| SC-002 | スケジュール解除 | タスクのスケジュールを解除 |
| SC-003 | 日別スケジュール表示 | 指定日のタイムライン形式でスケジュールを表示 |
| SC-004 | 週間スケジュール表示 | 週単位でスケジュールを一覧表示 |
| SC-005 | 日付ナビゲーション | 前日/翌日、前週/翌週への移動 |

#### スケジュール属性
- 予定日: 必須
- 開始時刻: 必須
- 終了時刻: 任意
- タイムスロット: 8:00〜21:00

## 3. 非機能要件

### 3.1 セキュリティ要件

| ID | 要件 |
|----|------|
| SEC-001 | パスワードはハッシュ化して保存 |
| SEC-002 | ログイン認証が必要なページは認証必須 |
| SEC-003 | ユーザーは自分のデータのみアクセス可能 |
| SEC-004 | CSRF対策を実装 |

### 3.2 ユーザビリティ要件

| ID | 要件 |
|----|------|
| UX-001 | 直感的なUI/UXを提供 |
| UX-002 | フラッシュメッセージで操作結果を通知 |
| UX-003 | 期限切れタスクを視覚的に強調表示 |
| UX-004 | 優先度を色分けで表示 |

### 3.3 性能要件

| ID | 要件 |
|----|------|
| PERF-001 | ページ読み込み時間: 3秒以内 |
| PERF-002 | 同時接続ユーザー数: 小規模利用を想定 |

### 3.4 運用要件

| ID | 要件 |
|----|------|
| OPS-001 | SQLiteデータベースを使用（ファイルベース） |
| OPS-002 | ローカル環境での実行を前提 |

## 5. 用語定義

| 用語 | 定義 |
|------|------|
| タスク | ユーザーが管理する作業単位 |
| サブタスク | タスクを細分化した子タスク |
| カテゴリ | タスクを分類するためのグループ |
| スケジュール | タスクに割り当てられた日時 |
| 優先度 | タスクの重要度（高・中・低） |
| 期限 | タスクの完了目標日 |
